What is Markdown?

Markdown is a plain text formatting syntax designed so that it can optionally be converted to HTML using a tool by the same name. Markdown is popularly used as format for readme files, or for writing messages in online discussion forums, or in text editors for the quick creation of rich text documents.1

History

The Markdown language was created in 2004 by John Gruber with substantial contributions from Aaron Swartz2

Markdown’s syntax has been influenced by several existing text-to-HTML filters — including Setext, atx, Textile, reStructuredText, Grutatext, and EtText — the single biggest source of inspiration for Markdown’s syntax is the format of plain text email.3

Sites such as GitHub, reddit, Diaspora, Stack Overflow, OpenStreetMap, and SourceForge use variants of Markdown to facilitate discussion between users.

Used in many sites for creation of content, like Hakyll, Ghost, Jakyll, DocPad, etc.

Pandoc Markdown

Pandoc is a Haskell library for converting from one markup format to another, and a command-line tool that uses this library.4

You can say that Pandoc is the markdown for technical writers/law documents/book writers/archivists/etc.

Writers & Extensions

Lua can be used as a writer to intercept and change the output.

  • document metadata (title, author, date)
  • footnotes
  • tables
  • definition lists
  • superscript and subscript
  • strikeout
  • enhanced ordered lists (start number and numbering style are significant)
  • running example lists
  • delimited code blocks with syntax highlighting
  • smart quotes, dashes, and ellipses
  • markdown inside HTML blocks
  • inline LaTeX

Extras

  • Editors have plugins for Pandoc (emacs, vim, Textmate, Sublime Text, etc.)
